# Contributing Guide

**Project:** Cross-platform audience reactions to the Break 50 × Trump episode  
**Course:** KIN 7518 — Social Issues in Sport

Welcome to the TSPS Group repository. This guide tells each group member
exactly where to work and what to avoid committing.

---

## ⚠️ The Golden Rule

> **Never commit raw comment data, cleaned CSVs containing comment text,
> or any file that includes usernames or user IDs.**
>
> If you are unsure whether a file is safe to commit, check the
> [safe-to-upload checklist](#safe-to-upload-vs-do-not-upload) below
> or ask the Data & Classification Lead before pushing.

## Where to Edit Each Type of Content

### Keyword Lexicons (`codebook/`)

- Edit `codebook/keywords.txt` to add, remove, or flag keywords in any of the four lists.
- After editing, re-run `scripts/02_classify.py` and then `scripts/03_descriptive_stats.py`
  to regenerate all tables.
- Commit the updated `keywords.txt` **and** the regenerated `tables/*.csv` together.

### Analysis Scripts (`scripts/`)

- Each script is numbered to reflect pipeline order. Edit the relevant script.
- After any script change, re-run from that script through the end of the pipeline
  (or just run `python run_all.py` to regenerate everything).
- Commit the edited script **and** any updated `tables/` or `figures/` files together.

### Summary Tables (`tables/`)

- Tables are **generated automatically** by `scripts/03_descriptive_stats.py`.
- Do not edit CSV files by hand. Edit the script or the keyword lists, then re-run.
- All eight table CSVs are safe to commit — they contain only aggregated counts,
  percentages, and engagement statistics with no comment text or usernames.

### Figures (`figures/`)

- Figures are **generated automatically** by `scripts/04_visualize.py`.
- Do not edit PNG files directly.
- Re-run `scripts/04_visualize.py` after any change to scripts or tables.
- Commit updated PNG files after regeneration.

### Documentation (`docs/`)

- `P3_PLAN_REVISED.md` — the full research plan. Any group member may edit this
  for clarity, but coordinate with the team before substantive changes.
- `methods_summary.md` — auto-generated by `scripts/05_methods_summary.py`.
  Do not edit by hand; re-run the script after table changes.

### README and Project Files (`/`)

- `README.md` — edit to reflect any major structural changes to the repo.
- `DATA_AVAILABILITY.md` — edit only if the data-sharing policy changes.
- `requirements.txt` — edit if you add a new Python dependency; add the package
  and a minimum version, e.g. `scipy>=1.11`.

## Git Workflow for Group Members

### One-time setup

```bash
# Clone the repository
git clone https://github.com/<your-org>/b50-comment-analysis.git
cd b50-comment-analysis

# Install dependencies
pip install -r requirements.txt
```

### Making changes

```bash
# Always pull the latest changes before you start
git pull origin main

# Make your edits, then check what changed
git status
git diff

# Stage only safe files — never stage data/raw/ or data/cleaned/*.csv
git add scripts/02_classify.py
git add codebook/keywords.txt
git add tables/

# Write a clear, specific commit message
git commit -m "feat: add 'legend' and 'iconic' to supportive lexicon; regenerate tables"

# Push to main (or to your feature branch — see below)
git push origin main
```

### Feature branches (recommended for larger changes)

```bash
# Create a branch for your work
git checkout -b feature/add-sarcasm-keywords

# ... make changes, add, commit as above ...

# Push your branch
git push origin feature/add-sarcasm-keywords

# Open a Pull Request on GitHub for team review before merging
```

## Safe-to-Upload vs. Do Not Upload

| File / Folder | Safe to commit? | Reason |
|---|---|---|
| `codebook/keywords.txt` | ✅ Yes | Keyword lists only — no user data |
| `tables/*.csv` | ✅ Yes | Aggregated statistics only |
| `figures/*.png` | ✅ Yes | Charts only — no user data |
| `scripts/*.py` | ✅ Yes | Analysis code |
| `docs/*.md` | ✅ Yes | Documentation |
| `run_all.py` | ✅ Yes | Pipeline runner |
| `requirements.txt` | ✅ Yes | Dependencies |
| `README.md`, `*.md` | ✅ Yes | Documentation |
| `data/raw/.gitkeep` | ✅ Yes | Empty placeholder |
| `data/raw/*.xlsx` | ❌ No | Raw comment data |
| `data/cleaned/B50_unified_comments.csv` | ❌ No | Row-level comment text |
| `data/cleaned/B50_classified_comments.csv` | ❌ No | Row-level comment text + usernames |
| Any file with a `user` or `text` column | ❌ No | Contains identifiable or raw data |
| `*.log`, `*.tmp`, `*.bak` | ❌ No | Scratch / generated files |
